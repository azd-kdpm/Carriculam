Title Slide - IP Addressing & Subnetting

  • Welcome to IP Addressing for security networks.
  • Learn how cameras and recorders find each other on network lines.
  • Understand IP addresses, subnets, and default gateways.
(12-2)

What is an IP Address?

  • An IP address is a unique digital address for every device.
  • IPv4 addresses use four numbers separated by dots (e.g. 192.168.1.10).
  • Every camera must have a unique IP address to communicate.
(12-3)

The Subnet Mask

  • Subnet mask defines which part of the IP is the network address.
  • Standard 255.255.255.0 mask allows up to 254 local devices.
  • Devices must be on the same subnet mask to talk to each other.
(12-4)

CIDR Classless Notation

  • CIDR uses a slash (like /24) instead of long subnet numbers.
  • 24 notation means 24 bits are locked for the network ID.
  • 28 notation creates smaller subnet groups for high security.
(12-5)

Private vs Public IP Ranges

  • Private IPs (192.168.x.x, 10.x.x.x) are used inside local networks.
  • Public IPs are assigned by internet providers for global access.
  • Private IPs cannot be accessed directly from the internet without setup.
(12-6)

Gateway & DNS Basics

  • The Gateway IP is the router address leading to the internet.
  • DNS translates web names into IP addresses for device lookup.
  • Set the NVR gateway to your router IP to enable remote viewing.
(12-7)

Static IP vs DHCP Dynamic

  • DHCP automatically gives IP addresses to phone and laptops.
  • IP cameras and NVRs require Static fixed IPs that never change.
  • Static IPs prevent video loss caused by IP address changes.
(12-8)

Sizing a CCTV Subnet

  • Group 30 cameras into a dedicated camera subnet range.
  • Keep room for future camera additions when picking subnet size.
  • Document all camera IP assignments on a site network sheet.
(12-9)

Common Configuration Errors

  • IP Address Conflicts happen when two cameras get the same IP.
  • Wrong subnet mask settings cause cameras to go offline.
  • Unplugged gateway lines stop mobile phone notifications.
(12-10)

Practical Lab Guide

  • Assign static IP addresses to two IP cameras manually.
  • Set subnet masks and router gateway parameters.
  • Ping camera IP addresses to test network connection speed.
