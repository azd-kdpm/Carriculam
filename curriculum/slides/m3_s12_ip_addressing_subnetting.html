Stage 3: Networking Fundamentals & Infrastructure

(13-2)

IPv4 Address Structure & Octets

  • 32-Bit Binary Address: Composed of 4 octets separated by dots (e.g. 192.168.1.100).
  • Decimal to Binary: Each octet ranges from 0 to 255 (8 bits per octet).
  • IP Uniqueness: Every device on an IP network must have a unique IP address to prevent conflicts.
Fundamental Rule: No two devices on the same local subnet can share the exact same IP address.
(13-3)

Subnet Masks & Host Ranges

  • Network vs Host: Subnet mask separates the network portion from the host portion of an IP address.
  • Default Class C Mask: 255.255.255.0 defines 254 usable host addresses (192.168.1.1 to 192.168.1.254).
  • Network & Broadcast IDs: First IP (.0) is Network ID; last IP (.255) is Broadcast ID (not usable for cameras).
Host Formula: Usable Hosts = 2^H - 2 (subtracting Network and Broadcast addresses).
(13-4)

CIDR Notation (/24, /27, /30)

  • Classless Inter-Domain Routing: Expresses subnet mask length by slash prefix (e.g. /24 = 255.255.255.0).
  • Subnetting Benefits: Subnetting a /24 network into smaller /27 subnets isolates camera traffic into 30-host groups.
  • Point-to-Point Links: /30 subnets (255.255.255.252) yield exactly 2 usable IPs for router-to-router links.
Quick Lookup: /24 = 254 hosts | /27 = 30 hosts | /30 = 2 hosts.
(13-5)

Private vs Public IP Address Ranges

  • Class A Private (10.0.0.0/8): Large enterprise networks (16 million host capacity).
  • Class B Private (172.16.0.0/12): Medium commercial building networks.
  • Class C Private (192.168.0.0/16): Standard small office and local CCTV camera networks.
Security Rule: Private IP addresses (10.x, 192.168.x) are non-routable on the public internet.
(13-6)

Static IP Reservation vs DHCP Auto-Leases

  • Static IP Assignment: Hardcoding fixed IP addresses on cameras and NVRs prevents connection drops.
  • DHCP Server: Automatically leases dynamic IP addresses to laptops and temporary technician tools.
  • DHCP MAC Reservation: Router locks specific IP addresses to camera MAC addresses in DHCP tables.
CCTV Standard: ALWAYS assign static IP addresses to IP cameras and NVR storage servers.
(13-7)

Default Gateway & Subnet Routing

  • Default Gateway: The IP address of the local router interface (e.g. 192.168.1.1) leading to other subnets.
  • Same-Subnet Traffic: Devices on 192.168.1.x communicate directly via Layer 2 switch without hitting the gateway.
  • Cross-Subnet Traffic: Traffic destined for a different subnet (e.g. 10.0.0.x) must be forwarded to the gateway router.
Configuration Step: Ensure camera default gateway matches router LAN IP to enable remote management.
(13-8)

MAC Address & Address Resolution Protocol (ARP)

  • MAC Address (Layer 2): 48-bit permanent hardware address (e.g. 00:1A:3F:F1:4C:88) assigned at factory.
  • OUI Vendor Code: First 3 bytes identify the camera manufacturer (e.g. Hikvision, Dahua, Axis).
  • ARP Table: Maps Layer 3 IP addresses to Layer 2 MAC addresses on switches and NVRs.
Identification Tip: Check the first 3 bytes of a MAC address to identify camera brand.
(13-9)

Network Diagnostic Commands (PING, TRACERT)

  • Ping (ICMP Echo): Tests basic IP connectivity and measures round-trip latency in milliseconds.
  • Tracert / Traceroute: Identifies all router hops and network latency bottlenecks along a path.
  • Arp -a: Displays resolved IP-to-MAC address mapping table on technician laptops.
Diagnostic Step: Run ping -t 192.168.1.100 to test for intermittent camera packet loss.
