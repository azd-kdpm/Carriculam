(21-2)

Spatial vs Temporal Compression Principles

  • Spatial Intra-Frame Compression: Compresses redundant pixels within a single image frame (similar to JPEG image compression).
  • Temporal Inter-Frame Compression: Compresses redundant pixels across sequential video frames by transmitting only moving pixel vectors.
  • Static Scene Savings: In static hallways, 95% of pixels remain identical frame-to-frame, allowing massive bitrate savings.
Compression Fact: Temporal compression avoids re-transmitting static background pixels every frame.
(21-3)

GOP (Group of Pictures) Structure: I-Frames & P-Frames

  • I-Frame (Intra-Coded Frame): Standalone full-detail reference image frame containing 100% complete image data.
  • P-Frame (Predictive Frame): Compressed delta frame containing only changes from the preceding I-Frame.
  • GOP Length (Keyframe Interval): Distance between I-Frames (e.g. GOP 30 = 1 I-Frame every 30 frames / 2 seconds at 15fps).
Configuration Rule: Match GOP Length to Camera Frame Rate (e.g. 15 FPS = GOP 30 for 2-second keyframe interval).
(21-4)

H.264 Advanced Video Coding vs H.265 High Efficiency

  • H.264 Fixed Macroblocks: Uses rigid 16x16 pixel macroblocks for motion estimation.
  • H.265 Coding Tree Units (CTU): Uses dynamic flexible 64x64 CTU blocks that shrink or expand based on detail.
  • 50% Bandwidth Cut: H.265 cuts storage and bandwidth consumption by 50% compared to H.264 at identical visual quality.
Codec Standard: Standardize on H.265 for all new IP CCTV installations to double HDD retention days.
(21-5)

Smart Codecs (Hikvision H.265+ / Dahua Smart H.265)

  • Dynamic Background Prediction: Extracts static background model from scene and updates only moving targets.
  • Enhanced Noise Suppression: Suppresses night-vision CMOS sensor noise without blurring moving human subjects.
  • 75%+ Storage Savings: Achieves up to 75% additional bandwidth savings over standard H.265 in typical surveillance scenes.
Smart Choice: Enable proprietary Smart H.265+ codecs to achieve up to 30 days storage on small hard drives.
(21-6)

Dynamic GOP Interval Adaptation

  • Static Night Adaptation: Camera automatically expands GOP length from GOP 30 up to GOP 300 when no motion is detected at night.
  • Instant Motion Shift: Camera instantly drops GOP length back to GOP 30 the millisecond a human or car enters the scene.
  • I-Frame Overhead Cut: Eliminates thousands of unnecessary heavy I-Frames during quiet night hours.
Night Savings: Dynamic GOP reduces night-time video file sizes down to under 300Kbps per camera.
(21-7)

Dynamic Smart ROI (Region of Interest) Encoding

  • Targeted Bitrate Allocation: Allocates high-resolution compression bits exclusively to critical target zones (e.g. license plates or faces).
  • Background Compression: Applies aggressive compression to non-essential areas like sky, empty walls, or grass.
  • Configurable Zones: Draw up to 4 fixed ROI rectangular zones inside the camera configuration web interface.
ROI Benefit: Keeps license plates razor-sharp while reducing background tree leaf bandwidth.
(21-8)

VBR vs CBR Bitrate Control for Variable Activity

  • VBR Mode (Variable Bitrate): Bitrate floats dynamically based on scene complexity; ideal for offices and retail stores.
  • CBR Mode (Constant Bitrate): Bitrate remains flat regardless of motion; mandatory for unstable wireless microwave links.
  • VBR Max Cap Sizing: Set VBR Max Limit to 4096Kbps to cap peak bandwidth while retaining static night savings.
Network Choice: Use CBR for wireless bridge links; use VBR with Max Cap for wired IP camera networks.

Storage Savings Math (Terabyte Reductions)

  • 16-Camera 30-Day Project: 16x 4MP cameras recorded continuously for 30 days.
  • H.264 Storage Requirement: Requires ~48 Terabytes of HDD storage (at 6Mbps per camera).
  • H.265+ Storage Requirement: Requires only ~12 Terabytes of HDD storage (at 1.5Mbps smart average bitrate).
Cost Impact: H.265+ cuts hard drive hardware costs from 4 HDDs down to 1 single HDD!
