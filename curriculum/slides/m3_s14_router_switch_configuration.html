(14-2)

Layer 2 Switches vs Layer 3 Routers

  • Layer 2 Switches: Forward Ethernet frames based on MAC addresses; connect devices inside the same VLAN.
  • Layer 3 Routers: Forward IP packets based on destination IP addresses; route traffic between different subnets.
  • Layer 3 Managed Switches: Combine high-speed wire-rate Layer 2 switching with hardware Inter-VLAN routing.
Core Difference: Layer 2 switches use MAC tables; Layer 3 routers use IP routing tables.
(14-3)

Managed Switch Access (Web GUI, CLI & Console)

  • Console Cable Access: Out-of-band RS232 / USB-to-RJ45 console connection for initial out-of-box setup.
  • Web Management GUI: User-friendly browser interface for port status, VLANs, and PoE power monitoring.
  • SSH / Telnet CLI: Command Line Interface for automated configuration scripting and enterprise management.
Security Standard: Disable insecure HTTP/Telnet and use HTTPS/SSH for switch management.
(14-4)

Port Speed, Duplex & Autonegotiation

  • Speed Ratings: FastEthernet (100Mbps) for IP cameras vs Gigabit (1000Mbps) for NVR and uplink ports.
  • Full-Duplex: Simultaneous bi-directional data transmission (sending and receiving at full bandwidth).
  • Autonegotiation Mismatch: Mismatched speed/duplex settings cause high packet collisions and video stutter.
Uplink Sizing: Ensure switch uplink ports to NVRs are minimum 1Gbps (Gigabit).
(14-5)

Router LAN & WAN Interface Configuration

  • LAN Interface: Internal gateway IP address connecting to local switches and security cameras.
  • WAN Interface: External interface connected to ISP modem (Static Public IP or PPPoE).
  • Network Address Translation (NAT): Translates internal private IPs (192.168.x.x) into a single public WAN IP.
NAT Purpose: Allows hundreds of internal IP cameras to share a single ISP public internet line.
(14-6)

Port Forwarding (NAT) & Virtual Servers

  • Port Mapping: Maps external public WAN ports to internal NVR service ports (e.g. Public 8000 → NVR 192.168.1.100:8000).
  • Standard Service Ports: HTTP Port 80, RTSP Port 554, and SDK/Media Port 8000.
  • Security Risk: Direct port forwarding exposes NVRs to internet brute-force attacks; VPN is preferred.
Security Warning: Always change default HTTP (80) and Media (8000) ports when port forwarding.
(14-7)

Quality of Service (QoS) Video Prioritization

  • DSCP / CoS Tagging: Tags video stream packets with high priority (e.g. DSCP 34 / AF41).
  • Congestion Prevention: Ensures video streams are not dropped when office PCs download large files.
  • Egress Queue Schedulers: Switches process high-priority video queues before low-priority web traffic.
QoS Benefit: Prevents camera video dropping during peak office network bandwidth usage.
(14-8)

Spanning Tree Protocol (STP / RSTP)

  • Loop Prevention: Prevents catastrophic broadcast storms caused by accidental redundant cable loops between switches.
  • RSTP Convergence (IEEE 802.1w): Rapid Spanning Tree block/forward state convergence in under 2 seconds.
  • Redundant Uplinks: Enables active-standby backup network links between core and edge switches.
Network Protection: ALWAYS enable RSTP on all managed switches to prevent loop crashes.
(14-9)

Network Topologies (Star vs Tree vs Ring)

  • Star Topology: All camera edge switches connect back directly to a central core switch (CCTV Standard).
  • Tree / Hierarchical Topology: Access switches connect to Distribution switches, which connect to Core.
  • Ring Topology (ERPS): Industrial fiber ring topology providing sub-50ms self-healing failover.
CCTV Standard: Star topology is recommended for minimum latency and simple troubleshooting.
