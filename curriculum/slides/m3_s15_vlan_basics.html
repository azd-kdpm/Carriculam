Stage 3: Networking Fundamentals & Infrastructure

(15-2)

Why Use VLANs? (Security & Isolation)

  • Broadcast Domain Reduction: Splits one physical switch into multiple isolated logical networks.
  • CCTV Security Isolation: Prevents office PCs and guest Wi-Fi users from probing camera streams.
  • Bandwidth Protection: Keeps heavy camera video multicast traffic off office computer networks.
Best Practice: ALWAYS isolate IP camera systems into a dedicated CCTV VLAN (e.g. VLAN 10).
(15-3)

IEEE 802.1Q Tagging Standard

  • 4-Byte VLAN Tag: Injects a 4-byte 802.1Q header into standard Ethernet frames containing the 12-bit VLAN ID (1-4094).
  • VLAN ID Range: VLAN 1 is default untagged; user VLANs range from VLAN 2 to VLAN 4094.
  • Frame Handling: Switches examine 802.1Q tags to route frames strictly to ports assigned to that VLAN ID.
Standard: IEEE 802.1Q is the universal multi-vendor standard for Ethernet VLAN tagging.
(15-4)

Access Ports vs Trunk Ports

  • Access Port (Untagged): Connects end devices (IP Cameras, NVRs, PCs); carries traffic for a single VLAN.
  • Trunk Port (Tagged): Interconnects switches and routers; carries traffic for multiple VLANs simultaneously using 802.1Q tags.
  • Native VLAN: Untagged traffic received on a trunk port is assigned to the designated Native VLAN (default VLAN 1).
Port Rule: Camera ports = Access Ports (Untagged); Switch-to-Switch links = Trunk Ports (Tagged).
(15-5)

CCTV Network Segmentation Architecture

  • VLAN 10 (IP Cameras): Dedicated subnet (10.10.10.0/24) for camera video streaming; no internet access.
  • VLAN 20 (Management & NVR): Subnet (10.10.20.0/24) for NVR servers, workstation clients, and VMS.
  • VLAN 30 (Corporate Data): Isolated corporate office network (192.168.1.0/24).
Security Standard: Block direct internet access on VLAN 10 (Camera VLAN) to prevent camera hacking.
(15-6)

Inter-VLAN Routing (Layer 3 Switch / Router)

  • VLAN Isolation: By default, devices on different VLANs CANNOT communicate with each other.
  • Inter-VLAN Router: Layer 3 switch or router routes traffic between VLANs based on Access Control Lists (ACLs).
  • ACL Firewall Rules: Allow NVR (VLAN 20) to pull video from Cameras (VLAN 10), but block Office PCs from Camera VLAN.
ACL Rule: Use Layer 3 ACLs to allow NVR access to cameras while blocking unauthorized users.
(15-7)

Broadcast Storm Control & Rate Limiting

  • Broadcast Storms: Faulty network cards or loop conditions flood ports with excessive broadcast packets.
  • Threshold Limiting: Sets maximum broadcast/multicast packets per second (e.g. 5% port bandwidth limit).
  • Port Suppression: Automatically drops excessive broadcast traffic when thresholds are exceeded.
Switch Protection: Enable Storm Control on camera ports to prevent network flooding crashes.
(15-8)

Port Security & MAC Address Locking

  • Physical Port Vulnerability: Outdoor cameras mounted on walls present accessible RJ45 network jacks.
  • Sticky MAC Security: Locks switch port to the specific camera's MAC address; shuts port if laptop is plugged in.
  • Violation Actions: Shutdown port, restrict traffic, or send SNMP trap alert to network administrator.
Outdoor Security: ALWAYS enable Port Security on outdoor camera switch ports to prevent unauthorized network access.
(15-9)

Multi-Switch Trunk Link Configurations

  • 802.1Q Trunk Setup: Configure switchport mode trunk and allowed VLAN 10,20 on switch uplinks.
  • Fiber Optic Uplinks: Use SFP/SFP+ fiber transceivers to connect building access switches to core switches.
  • Link Aggregation (LACP): Bundles two 1Gbps uplinks into a 2Gbps redundant trunk link.
Bandwidth Sizing: Use LACP port channeling for switch uplinks carrying more than 30 IP camera feeds.
