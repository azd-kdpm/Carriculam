Stage 5: Network & Camera Troubleshooting

(28-2)

The Diagnostic Methodology (Layer-by-Layer OSI Flow)

  • Bottom-Up Approach: Start at Layer 1 (Physical Cable) before inspecting Layer 3 (IP Routing).
  • Layer 1 Physical Check: Verify Ethernet link LED lights, RJ45 crimp pins, and PoE power status.
  • Layer 2 / 3 Verification: Check MAC address tables on managed switches and IP gateway ping reachability.
Golden Rule: 80% of network failures are caused by Layer 1 physical cable damage or bad RJ45 crimps.
(28-3)

ICMP Ping Diagnostics & Packet Loss Analysis

  • Continuous Ping (ping -t): Tests continuous network connectivity and flags intermittent frame drops.
  • Buffer Size Testing (ping -l 1472): Sends maximum non-fragmented MTU payload to detect MTU size mismatches.
  • Latency Jitter Thresholds: Latency spikes > 50ms cause RTSP video artifacting and frame freezing.
CLI Tip: Use "ping 192.168.1.100 -t -l 1472" to test full-size MTU packet delivery to IP cameras.
(28-4)

Traceroute & Path Latency Isolation

  • Hop-by-Hop Tracing (tracert): Displays every router hop along the network path to destination IP.
  • TTL Exceeded Feedback: Uses ICMP TTL expiration to pinpoint exact router or firewall blocking video traffic.
  • Bottleneck Identification: Identifies overloaded wireless microwave links or congested switch uplinks instantly.
CLI Tip: Use "tracert" when cameras fail to stream across multi-subnet corporate WAN routers.
(28-5)

IP Address Conflicts & ARP Table Inspection

  • Duplicate IP Conflict: Occurs when two devices are assigned the same static IP, causing flickering video streams.
  • ARP Table Inspection (arp -a): Displays IP-to-MAC address mappings cached on the computer or NVR.
  • Flapping MAC Symptoms: If ARP entry MAC address changes rapidly between two MACs, an IP conflict exists.
Troubleshooting Fix: Run "arp -a" to detect duplicate IP conflicts when an NVR camera stream repeatedly drops.
(28-6)

Subnet Mask Mismatches & Default Gateway Faults

  • Subnet Boundary Errors: Camera configured with /24 (255.255.255.0) cannot talk to VMS on /23 (255.255.254.0).
  • Missing Gateway Address: Camera missing Default Gateway IP cannot transmit video to remote mobile apps over Internet.
  • Broadcast Domain Isolation: Mismatched subnets prevent WS-Discovery multicast search from locating cameras.
Network Rule: Verify IP address, Subnet Mask, and Gateway are in identical network ranges.
(28-7)

Wireshark Packet Capture & RTSP Protocol Analysis

  • Network Packet Sniffing: Captures real-time Ethernet frames passing through network switch mirror ports.
  • RTSP Stream Inspection: Filters by "rtsp" or "rtp" to verify OPTIONS, DESCRIBE, SETUP, and PLAY RTSP handshakes.
  • Authentication Failures: Inspects 401 Unauthorized response headers to detect invalid ONVIF credentials.
Sniffer Tip: Use Wireshark filter "rtsp || onvif" to diagnose camera streaming connection failures.
(28-8)

Switch Port Mirroring (SPAN Port Configuration)

  • SPAN (Switched Port Analyzer): Copies all ingress/egress traffic from target camera port to diagnostic monitoring port.
  • Non-Intrusive Sniffing: Allows engineers to inspect live camera traffic without disconnecting active cables.
  • Managed Switch Setup: Configured via switch CLI (monitor session 1 source interface gi1/0/5).
Diagnostics Rule: Configure SPAN port mirroring on managed switches to capture full-speed camera traffic.
(28-9)

Port Scanner Tools (Angry IP Scanner & Nmap)

  • IP Network Sweeps: Angry IP Scanner rapidly scans IP ranges (e.g. 192.168.1.1-254) to discover active devices.
  • Open Port Auditing: Scans key surveillance ports (Port 80 HTTP, 554 RTSP, 8000 SDK, 3702 WS-Discovery).
  • Firewall Block Verification: Identifies if router ACLs or Windows Firewall are blocking Port 554 RTSP video traffic.
Tool Tip: Use Nmap "nmap -p 80,554,8000 192.168.1.100" to verify open CCTV ports.
(28-10)

Practical Network Diagnostics & Wireshark Lab

  • Task 1: Run continuous MTU size ping test (ping -l 1472) to test packet fragmentation on switch link.
  • Task 2: Inspect ARP table (arp -a) to identify duplicate IP address conflict between test devices.
  • Task 3: Configure SPAN port mirror on managed switch and capture RTSP 200 OK handshake in Wireshark.
Lab Goal: Master ping MTU testing, ARP conflict resolution, and Wireshark RTSP packet capture.
