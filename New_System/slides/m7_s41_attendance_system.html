Push SDK vs Pull Protocol Communication

  • Legacy Poll/Pull Protocol: Central server periodically polls terminals over IP, causing network congestion and delayed logs.
  • Modern HTTP/HTTPS Push SDK: Terminals instantly push biometric punch events to cloud server within 0.1 seconds.
  • Firewall & NAT Traversal: Push SDK requires zero port forwarding or public static IPs at remote branch offices.
Architecture Standard: ALWAYS deploy HTTP Push SDK biometric devices for multi-branch corporate attendance.

Shift Roster Engines: Fixed, Rotational & Flexible

  • Fixed Shifts (9 AM - 5 PM): Standard corporate workday schedules with fixed punch windows.
  • Rotational Shifts (24/7 Operations): Auto-detects Morning, Evening, and Night shift punches (e.g. 8-hour rotating shifts).
  • Overnight Shift Math: Intelligently pairs In-punch (10 PM) and Out-punch (6 AM next day) into single working shift.
Software Feature: Auto-shift detection matches punches to the closest active shift roster automatically.

Attendance Rules: Grace Periods, Late-In & Early-Out

  • Grace Period Tolerance: Configures 15-minute grace period (e.g. punches up to 09:15 AM marked ON-TIME).
  • Late-In / Early-Out Rules: 3 Late-In punches automatically trigger half-day salary deduction.
  • First-In / Last-Out Calculation: Filters intermediate punches throughout the day to calculate net working hours.
HR Rule: Grace period rules avoid unnecessary salary penalties for minor traffic delays.

Troubleshooting Push Server Disconnections & Missing Logs

  • Push Server Offline (Terminal Caching): Terminals store up to 100,000 logs offline; auto-resyncs when server connection restores.
  • Mismatched Device ADMS Settings: Verify IP address, port (e.g. 8081), and communication key in device menu.
  • Duplicate User ID Conflict: Ensure unique User IDs across all branch terminals to prevent record overwrites.
Troubleshooting Tip: Biometric devices store logs offline safely during network outages and auto-sync upon reconnection.
