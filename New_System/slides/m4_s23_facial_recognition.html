Facial Recognition Workflow

  • Step 1 (Face Detection): Locates human face boundaries in the video stream using deep learning.
  • Step 2 (Alignment & Normalization): Aligns eye/nose geometry and adjusts lighting contrast.
  • Step 3 (Feature Extraction): Converts facial landmark geometry into a unique mathematical vector string.
  • Step 4 (Database Match): Compares generated vector against enrolled database templates in under 100ms.
Process Rule: Vector matching compares mathematical numbers, NOT actual JPEG image files.

(24-4)

Biometric Feature Vector Extraction

  • Landmark Node Network: Maps 68 to 128 unique nodal points (eye corners, nose tip, jawline, lip contours).
  • Irreversible Vector Hash: Encodes nodal distances into a 512-byte encrypted mathematical template.
  • Template Security: Original face image CANNOT be reconstructed from the stored mathematical vector hash.
Privacy Protection: Storing vector hashes instead of raw photos protects biometric data privacy.
(24-5)

1:1 Authentication vs 1:N Identification Matching

  • 1:1 Verification (One-to-One): Compares live face against single enrolled user ID (e.g. Access Control Card + Face at turnstile).
  • 1:N Identification (One-to-Many): Compares live face against entire database of 100,000+ faces in under 200ms.
  • Similarity Threshold (e.g. 80%): Adjusts matching confidence score threshold to balance false acceptance vs false rejection.
System Choice: Use 1:1 verification for high-security turnstiles; use 1:N identification for VIP detection.
(24-6)

Face Database Management (VIP, Staff, Blacklist)

  • VIP Greeting List (Green): Triggers instant notification to hotel manager when VIP guest enters lobby.
  • Employee Access List (Blue): Grants hands-free access control entry through secure turnstile gates.
  • Blacklist Security Alert (Red): Triggers immediate silent alarm on security guard consoles when barred persons enter.
Database Hygiene: Regularly audit facial databases to remove expired temp employee templates.
(24-7)

Environmental Lighting Requirements (300 Lux)

  • 300 Lux Even Lighting: Requires minimum 300 Lux even illumination across subjects' faces without harsh shadows.
  • Backlight Shadow Fault: Bright glass entrance doors behind visitors create dark facial silhouettes that break recognition.
  • WDR & Auxiliary Fill Light: Deploy WDR cameras or soft 3000K LED warm fill lights at dark entrance vestibules.
Lighting Rule: Install soft auxiliary fill lighting at glass entrance doors to eliminate backlight shadows.
(24-8)

Liveness Detection & Anti-Spoofing Protection

  • 2D Photo Screen Spoof Attack: Intruders holding up smartphone photos or printed paper faces to fool basic cameras.
  • 3D Binocular Stereo Liveness: Dual-lens depth sensing verifies 3D nose bridge, eye blinking, and micro-expressions.
  • IR Texture Analysis: Infrared sensor detects living skin thermal reflectivity versus flat paper/screen reflection.
Anti-Spoofing Standard: Mandatory 3D binocular stereo liveness detection on all access control face terminals.
(24-9)

GDPR Biometric Privacy & Data Encryption

  • Explicit User Consent: Compliance rules mandate clear signage and user consent before collecting facial biometrics.
  • AES-256 Vector Encryption: Encrypts facial database templates at rest and in transit over HTTPS.
  • Automated Privacy Masking: Automatically blurs non-target bystander faces in public video exports.
Compliance Rule: Encrypt biometric databases using AES-256 and display clear privacy notice signs.
