Hardware RAID Controllers vs Software RAID

  • Dedicated RAID ASIC Processor: Offloads parity calculations from host CPU to dedicated hardware controller.
  • Battery Backup Unit (BBU / Flash Cache): Protects unwritten write-back cache memory during sudden building power outages.
  • Software RAID Limitations: Software RAID (Windows Dynamic Disks) consumes server CPU and risks array corruption during OS crashes.
Hardware Rule: ALWAYS install a dedicated hardware RAID controller with Battery Backup (BBU) for surveillance servers.
(19-3)

RAID 5 Parity Striping (Single Drive Fault Tolerance)

  • Block-Level Parity Striping: Data blocks and parity checksums are distributed across a minimum of 3 physical drives.
  • Single Drive Failure Tolerance: Array continues operating seamlessly if any 1 single hard drive fails completely.
  • Usable Storage Math: Usable Capacity = (N - 1) Ã— Drive Size (1 drive capacity dedicated to parity protection).

RAID 6 Dual-Parity (Two Drive Failure Protection)

  • Dual Reed-Solomon Parity: Writes two independent parity blocks (P + Q) across a minimum of 4 physical drives.
  • Dual Drive Failure Tolerance: Array survives 2 simultaneous drive failures without losing a single frame of video evidence.
  • Usable Storage Math: Usable Capacity = (N - 2) Ã— Drive Size (2 drives capacity dedicated to dual parity).
Enterprise Standard: Use RAID 6 for storage arrays exceeding 8 drives to protect against dual-drive failure during rebuilds.
(19-5)

RAID 10 Striped Mirroring (Maximum Performance)

  • Striped Mirror Pairs (RAID 1 + RAID 0): Combines high-speed data striping with 100% drive mirroring across 4+ drives.
  • Maximum Read/Write Speed: Highest IOPS write performance for heavy 64-channel 4K video recording workloads.
  • 50% Storage Overhead: Exactly half of total raw storage is consumed by 1:1 drive mirroring.
Performance Choice: Deploy RAID 10 for high-security VMS servers requiring maximum write IOPS and fast rebuild times.
(19-6)

Hot-Spare Standby Drives & Auto-Rebuilding

  • Global Hot-Spare: Powered-on unassigned drive waiting in standby inside server chassis bay.
  • Automatic Fault Detection: Upon drive failure, RAID controller instantly powers up Hot-Spare and initiates auto-rebuild.
  • Zero Technician Delay: Eliminates manual drive replacement delay, reducing vulnerable array window from days to hours.
Reliability Standard: ALWAYS configure at least 1 Global Hot-Spare drive in every NVR storage enclosure.
(19-7)

SAN (Storage Area Network) vs NAS (Network Attached Storage)

  • SAN Storage (Block-Level / iSCSI & Fiber): Formats storage targets as raw block drives over dedicated 10GbE / Fiber Channel links.
  • NAS Storage (File-Level / NFS & SMB): Shares network file folders across standard office Ethernet networks.
  • VMS Compatibility: Enterprise VMS applications require block-level SAN iSCSI targets for high-speed multi-stream video writing.
VMS Storage Rule: Use block-level iSCSI SAN expansion units for enterprise camera storage expansion.
(19-8)

Mean Time Between Failures (MTBF) & Rebuild Stress

  • Surveillance Enterprise MTBF: Enterprise drives feature 2.5 million hours MTBF rating rated for 24/7 continuous operation.
  • Unrecoverable Read Error (URE): High-capacity drives (18TB+) risk URE read errors during multi-day RAID 5 rebuilds.
  • Rebuild Stress Protection: Using RAID 6 prevents array collapse if a second drive develops bad sectors during rebuild.
Drive Selection: NEVER use desktop drives in NVR storage arrays; use enterprise surveillance drives (WD Purple Pro / SkyHawk AI).
(19-9)

Hot-Swappable Enclosure Bays & Status Indicators

  • Hot-Swappable Drive Trays: Replace failed hard drives while NVR server remains running without powering down.
  • LED Status Codes: Solid Green = Normal Array; Flashing Amber = Active Rebuilding; Solid Red = Drive Failure.
  • Enclosure Management: Backplane SAS expanders report fan speeds, temperature sensors, and drive slot status via IPMI.
Maintenance Safety: Always check drive tray slot numbers before pulling a failed drive to prevent pulling the wrong healthy drive.
